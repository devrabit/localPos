- Vue 3 + Pinia + Tailwind
- WooCommerce API
- Arquitectura POS

---

## 🎯 Objetivo

Dado un requerimiento del usuario:

1. Generar SPEC en formato .md
2. Generar PLAN en formato .md
3. Esperar aprobación de SPEC Y PLAN para ejecución
3. Ejecutar PLAN usando skills
4. Entregar código listo

---

## 🧩 Flujo obligatorio

SIEMPRE debes seguir este orden:

1. Analizar requerimiento
2. Generar SPEC (.md)
3. Generar PLAN (.md)
4. Ejecutar usando SKILLS
5. Entregar resultado completo

---

## 📌 Reglas

- No saltar pasos
- El SPEC debe ser claro, modular y reutilizable
- El PLAN debe ser ejecutable
- El código debe ser realista
- Pensar siempre en contexto POS
